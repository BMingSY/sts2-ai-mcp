## 近期 A10 连续失败总结

近期连续失败的共同主因不是路线把生命耗尽、药水保留或 MCP 执行异常，而是构筑时过度重视“有协同”和理论上限，低估最坏常见手牌的独立生存能力。

- 多副牌在进入致死遭遇时仍有足够生命，说明面板 HP 不是主要解释。
- 新牌大多能加强某个引擎，却同时稀释即时防御、核心访问率和费用转化；“每张牌都有协同”不等于整副牌稳定。
- 多个依赖能力、触发牌或自伤的组件被误算成多条防线。核心沉底或牌序错开时，它们会同时失效。
- 牌组能制造高爆发或完整能力上限，却无法连续覆盖普通坏牌序与多个高压回合。
- 反复休息通常是战斗不稳定造成的后果；它又会减少升级机会，形成强度与生命互相拖累的循环。

下局纪律：每张新增牌先回答“它是否改善最坏常见手牌中的即时生存、核心访问或明确斩杀时钟”。如果只增加已有协同而不修复缺口，默认跳过。

## 既有失败模式

| 失败模式 | 可见征兆 | 可复用结论 |
| --- | --- | --- |
| 过度保住当前 HP，忽略取消未来意图的阈值 | 防御回合很好看，但敌人仍顺利进入强化攻击 | 击晕、转阶段和破盾阈值都是防守资源；比较整段循环后的伤害，而不是单回合掉血 |
| 随机或自动攻击成为唯一主轴 | 普通敌人的荆棘、复活单位或优先目标能关闭主要输出 | 自动攻击只能是增益，不能承担必须命中的任务；需要定向替代线 |
| 自伤组件很多，独立即时防御很少 | 输出成长充足，但核心能力沉底时第一循环没有可支付格挡 | 自伤成长必须同时补即时防御、访问和有效生命预算 |
| 条件防御被误算成多条防线 | 能量、能力和触发牌都在，但错开上手后无法连续攻防 | 一条依赖链只算一条答案；至少保留一层不依赖主引擎的防御 |
| 慢引擎没有访问率和斩杀时钟 | 高生命进入成长 Boss，仍拖到敌人治疗、力量或限制失控 | 核心必须在首个危险循环前部署，或有不依赖它的替代线 |
| 多目标遭遇只追求对称压血 | 随机目标或朝向变化让同步击杀失去保证 | 同步压血只是手段；先保证未来意图下的生存和确定定向能力 |
| 为商店或未来奖励锁进长风险段 | 可靠恢复前连续出现必经战斗、战斗型未知或精英 | 选路追踪到真实恢复/转向点，并检查该点之后立即锁定的房间 |
| 药水等到牌组已无转化能力时才用 | 低血终局才交药，仍需高掷特定牌才能活 | 当前房间可能终结运行时，在仍能转化为伤害、抽牌或防御的窗口用药 |

## 低进阶成功模式

- 成功构筑同时具备定向前置输出、牌流、成长和防御闭环，而不是只完成一个高上限引擎。
- 厚牌组只有在抽牌、低费、能量、消耗或保留足以补偿时才稳定；成功样本不能证明“多拿牌无成本”。
- 低进阶成功只用于提出机制与构筑假设，不能直接推广成 A10 固定卡牌评级。

## 跨样本结论

- 最常见失败不是终局少一张防御，而是牌组在普通坏牌序或常见机制面前只有单一出口。
- 连胜构筑要降低依赖链：即时防御、访问工具、成长防御和斩杀时钟至少有部分彼此独立。
- “避开所有精英”和“为了成长硬吃所有精英”都不是稳定策略；精英必须通过准备度与后续出口检查。
- 药水保存价值必须与当前死亡概率比较。当前房间若可能结束运行，未来 Boss 价值不能成为不用药的理由。
- 复盘应找最早显著降低完整通关概率的不可逆决策；终局只用于判断是否仍有出口，不记录无复用价值的具体出牌流水。

## 新增复盘格式

后续只追加匿名、代表性的模式摘要，并记录：

1. 游戏/Mod/协议版本、角色与进阶。
2. 失败或成功发生在哪类阶段与遭遇，不记录 run ID 或种子。
3. 终局是否已经锁死。
4. 最早主要分歧及当时可见的功能缺口。
5. 失败类型：机制误读、执行、构筑、路线、资源时机或正常方差。
6. 一个可验证的下局纪律。

不要记录逐回合出牌顺序、完整牌组流水、可识别的日志标识或依赖隐藏信息的事后路线。只有会改变未来同类决策的机制结论才进入 skill；具体证据保留在原始日志中。
