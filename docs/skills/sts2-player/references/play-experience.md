# 战士 A10 NOSL 默认通关策略

本文件是用户要求 Codex 游玩战士 A10 时的默认策略。目标只有一个：在不存读档重试的前提下，击败第三层连续两个 Boss 并取得真实胜利结算。

> 最近审阅：2026-07-23
> 验证基线：游戏 v0.107.1；Mod 0.1.8；protocol `2026-07-18-v2-draft`；state 13；decision 6。
> 所有数值、机制和卡牌文本以当局 live state 与 `lookup_game_data` 为准。

## 证据权重

按下列顺序使用证据，后者不能覆盖前者：

1. 当前 A10 的 live 状态、已知 Boss、牌组、遗物、药水和路线。
2. A10 重复失败模式与推进最远的 A10 样本。
3. 当前版本的卡牌、敌人和 power 数据。
4. A0/A5 胜局只用于发现可能的协同，不用于证明 A10 强度。

低进阶胜局没有经历 A10 的数值压力与第三层双 Boss 检查。不得为了复刻低进阶胜局硬拿卡、硬走路线或把某个流派视为已验证答案。

## A10 的四项硬验收

一副牌必须同时通过四项验收；只有理论上限不算通过。

| 验收项 | 通过标准 | 未通过的典型表现 |
| --- | --- | --- |
| 启动 | 主攻防在首个危险循环前可访问、可支付；核心沉底时也有替代线 | 能量或能力很多，但坏起手没有可打的攻防 |
| 防御 | 有即时防御，也有能覆盖连续高压的成长/循环防御 | 只挡一次峰值；或能力与触发牌错开便一起失效 |
| 时钟 | 能在敌人治疗、加力量、塞状态或永久减益失控前结束战斗 | 高血进入 Boss，十余回合后仍只有防守、没有收束 |
| 匹配 | 有多目标/定向减员、成长战和第三层双 Boss 的答案 | 随机目标、复活、伤害限制或关键小怪关闭唯一输出 |

HP、药水、厚度和“牌都很强”都不能替代这四项验收。

## 默认构筑政策

### 先补功能，再谈流派

每次拿牌、买牌或换牌前，维护五个字段：

- `frontload`：当前前置输出怎样快速减员或越过早期机制。
- `defense`：核心沉底时仍可用的即时防御，以及长战防御怎样增长。
- `scaling`：唯一的主要伤害成长方式。
- `access`：抽牌、能量、保留、牌顶或消耗怎样让核心及时出现并支付。
- `matchup_gap`：下一个精英/Boss 尚未解决的机制。

一张新牌必须明确填入其中一个空缺或显著加强已有主轴。说不清槽位，默认跳过。

### A10 默认拿牌门槛

近期 A10 失败样本几乎不跳牌，却反复保留大量基础牌并缺少升级。新政策把“跳过”设为中后期默认：

1. 第一层前半可拿少量独立生效的前置输出与即时防御，先活过普通战和第一个精英。
2. 当前输出已经能处理下一段路线后，不再因为卡本身优质或“以后可能协同”继续拿同功能牌。
3. 第一层末确定一个主要伤害成长和一个主要防御引擎。第二层起，除修复明确匹配外，不再开启新的慢速主轴。
4. 需要另一张尚未拥有的牌才能工作，默认跳过；已有搭档但启动仍不稳定，也先补访问而不是补第三个组件。
5. 新增攻击、能力或触发器的速度不得持续快于抽牌、能量、压缩和独立防御的增长。

牌组不设机械张数上限。判断的是有效牌密度：在核心沉底、混入固定灾厄/状态、且抽到一张基础牌的常见坏手中，仍能否完成当回合攻防。

### 当前候选骨架，不是固定答案

A10 尚无胜局，因此没有任何流派被视为已验证最优。若奖励自然支持，优先考虑依赖较少、能同时改善多个验收项的骨架：

- 消耗防御：`NO_PAIN` 配合 `SECOND_WIND`、`TRUE_GRIT`、`BURNING_PACT` 等，把废牌压缩与防御合并。
- 攻击转防御：`RAGE` 让前置输出在高压回合同时产生格挡，但必须有足够可支付攻击和访问。
- 单一力量/易伤时钟：如 `WITH_ME`、`INFLAME`、`SHOCKWAVE`、`UPPERCUT` 等，让现有攻击共同成长，避免另开独立伤害包。
- 访问与能量：如 `OFFERING`、`BATTLE_TRANCE`、`POMMEL_STRIKE`、`PILLAGE`、`BLOODLETTING`；它们的价值取决于手牌能否把资源转化成当回合攻防。
- 独立保险：高效即时格挡、弱化/降力量、紧急大格挡或药水，防止主能力沉底即死亡。

这套骨架只是目前最值得继续检验的候选，因为它能把压缩、防御、输出和访问连接起来。只有当局已经具备对应组件并通过四项验收时才继续投入。慢速格挡能力、自伤能力、随机自动攻击或多套成长包都可以赢，但不得在缺少启动与独立防御时作为主计划。

### 基础牌、移除和升级

- 早期新增牌解决生存后，提高移除基础攻击、基础防御、固定死牌和低影响边际牌的优先级。
- 第一层 Boss 前没有任何核心升级是准备不足警报。除非休息确实跨过下一战死亡阈值，否则优先取得至少一次能明显改善启动、费用、抽牌或关键攻防阈值的升级。
- 连续休息是牌组或路线不稳定的证据，不是长期解法。若恢复持续挤占升级，下一段路线应降风险，奖励选择应停止扩张组件。
- 商店先修单点故障：关键移除、访问、即时防御、药水保险或能立即闭合主轴的组件。便宜但边际的牌仍会占抽牌位。

## 分阶段执行

### 第一层：获得独立战斗力并收敛

- 先获得能独立生效的前置输出；不能等两张能力同时出现后才会打伤害。
- 在第一个精英前检查前两回合：既要能减员，也要能支付防御，并有一瓶药水或等价保险覆盖坏手。
- 火堆默认优先升级能改变启动或关键阈值的核心牌；只有回复明确跨过下一段死亡区间才休息。
- 第一层末写定 `scaling` 与 `defense`。仍有两个以上未完成的半成品主轴，视为构筑未收敛。

### 第二层：证明最坏手牌和机制覆盖

- 停止“看到好牌就拿”，优先补访问、连续防御与定向多目标能力。
- 对复活/接续、多敌人成长、朝向、随机目标和成长 Boss 做 live 查询；不能用面板伤害替代机制答案。
- 第二层 Boss 前必须证明：核心能力沉底时能活过首个危险循环；Boss 的治疗/力量/永久减益失控前有真实斩杀时钟。
- Boss 提供限制出牌、抽牌、能量或自伤的选择时，先查询精确 power。会直接关闭当前引擎的选项不能只因平均代价较小而选择。

### 第三层：按两场 Boss 预算，不按一场验收

- 第三层不再主动开启新主轴；新增牌主要修复已知 Boss 匹配、启动和独立保险。
- 进入首个 Boss 前做双场预算：第一场需要多少 HP、药水和一次性资源，战后固定回复后第二场还剩多少有效生命与保险。
- 第一个 Boss 的通过标准不是“能赢”，而是常见坏牌序下赢完仍能承受第二个 Boss 的首个危险循环，并保留至少一条启动或药水兜底。
- 如果预计第一场会耗尽药水、让有效生命跌入第二场早期攻击区间，当前牌组仍未通过 A10 终局验收；应在此前的商店、火堆和奖励中修复启动或时钟。
- 第二场有关键小怪时，必须有定向快速减员；纯随机伤害不能算答案。面对永久减益或敌方持续成长时，防御引擎必须配套更快的收束速度。

## 路线与资源政策

### 路线

1. 先看已知 Boss、所有可达火堆/商店、强制精英和汇合点。
2. 对候选路线追踪到下一个真实恢复或转向点，计入普通战、可能转战斗的未知、强制精英和后续立即锁定的房间。
3. 精英只有在前两回合攻防、关键机制、药水保险和战后出口都通过时才进入。高 HP 不能代替准备度。
4. 也不能为了保血回避全部精英；若缺少终局成长，需要选择通过准备度检查的精英换取强度。

### HP 与自伤

- HP 是完整路线和双 Boss 的风险预算，不是当前分数，也不是可无限兑换的资源。
- 自伤、敌方攻击、荆棘、回合末伤害和下回合开始自动掉血分别计算；格挡不能抵扣“失去生命”。
- `[当前机制 v0.107.1]` “失去生命”可以把角色从 1 HP 降到 0；`playable=true` 只表示合法，不表示打出后能活。
- 部署持续自伤能力前，计算在无额外回复时还能触发几次，并证明斩杀时钟更短。不能用战后治疗支付当前战斗的生命。

### 药水

- 药水用于消除失败尾部。当前房间可能结束运行时，在牌组仍能把药水转化为生存或斩杀的窗口使用。
- 已有安全抽牌/生成动作时，先获取信息再决定是否喝静态增益药；但不能拖到只剩高掷随机药水才能活。
- 第三层为两场 Boss 分配资源，但第一场若不使用就无法稳定通过，先保证存活；随后把资源耗尽记为牌组准备不足，而不是用药错误。

## 战斗决策协议

每个回合依次执行：

1. 汇总敌方攻击、回合末 power、自伤、荆棘/反伤、复活、转阶段和自动药水；不要只读 `incoming_damage`。
2. 检查确定斩杀、击晕、转阶段或击杀关键目标是否比堆格挡取消更多未来伤害。
3. 找出一条确定生存线，再比较抽牌、生成或随机线路；已有安全线时不为小幅期望收益改赌。
4. 计算抽牌后的剩余能量。抽到答案却无费用支付，不是有效出口。
5. 结束回合前重算本回合未格挡伤害与所有回合末/下回合开始触发。

陌生机制一旦会改变生死、目标优先级或不可逆选择，必须先查 live state、`lookup_game_data` 与已有 MCP 日志。只能确认文字、不能确认结算顺序时，选择不依赖该假设的安全线。

## 每次不可逆决策的简短记录

客户端理由只需记录可复用的状态，不写长篇叙事：

- 当前 `scaling / defense / access / matchup_gap`。
- 本选择修复哪个缺口；若拿牌，说明它在核心沉底时的作用。
- 选路说明下一个真实恢复/转向点和强制风险段。
- Boss 前说明四项验收与双 Boss 资源预算是否通过。

## 复盘与策略演化

1. 先判断终局是否已有合法生存/斩杀线，再向前找最早显著降低完整通关率的不可逆决策。
2. 区分机制误读、执行、构筑、路线、资源时机、技术异常和正常方差。技术异常必须由 MCP/Mod 日志证明，不能用来解释正常 `GAME_OVER`。
3. 同一失败模式重复出现才升级为默认规则；单局现象只作为待检验假设。
4. 下一批 A10 尝试保持核心拿牌门槛与阶段验收稳定，只针对已证实的失败模式改一项规则，避免每局更换整套理论而无法学习。
5. 只把匿名、跨局可复用结论写入 skill；run ID、种子和逐回合操作保留在原始日志。
