# STS2 匿名实战复盘模式

本文件保存支持策略演化的匿名模式，不承担逐步战报。原始证据位于 `<repo>/agent_knowledge/`；不得把 run ID、种子、完整牌组流水或逐回合出牌顺序复制进 skill。

> 最近审阅：2026-07-23。
> 证据边界：低进阶胜局是弱对照；A10 失败只能证明现行策略未过关，不能单独证明某套替代牌组必胜。

## 本轮证据范围

本轮审阅了两份可确认的低进阶胜局，以及十三份具有明确终局的战士 A10 失败日志。中断、未完成或疑似技术异常的运行不计入战略结论。

A10 样本分布覆盖第一层、第二层普通/精英、第二层 Boss 和第三层 Boss；其中有三局到达第三层终局，一局击败首个第三层 Boss 后死于第二个 Boss。失败分布足以识别重复模式，但尚不足以宣称某个流派已在 A10 获得验证。

## 胜负对照矩阵

| 样本阶段 | 重复现象 | 能支持的结论 | 不能支持的结论 |
| --- | --- | --- | --- |
| A0/A5 胜局 | 牌组形成了访问、攻防转换和伤害成长闭环；终局只需打一场 Boss | 闭环与资源转化值得作为候选协同 | 该牌组照抄即可通过 A10 |
| A10 第一层失败 | 奖励几乎全拿、基础牌未处理、核心升级不足；高生命也会死于成长战 | 第一层必须获得独立战斗力并在层末收敛 | 多拿牌只是无害的早期探索 |
| A10 第二层普通/精英失败 | 多目标、复活/接续或坏起手令条件引擎同时失效 | 需要定向减员、独立即时防御和访问冗余 | 只要平均输出高就能靠操作弥补 |
| A10 第二层 Boss 失败 | 接近满血入场仍因机制选择、永久减益和斩杀时钟不足被拖死 | HP 不是主因；Boss 机制与时钟必须在入场前验收 | 多休息即可解决构筑问题 |
| A10 第三层首 Boss 失败 | 满血或高血入场仍被启动慢、永久减益或成长速度击穿 | 到达第三层不代表终局强度合格 | 能挡若干回合就等于有稳定防御 |
| A10 第二个第三层 Boss 失败 | 首 Boss 虽胜但消耗大量生命/药水，第二战早期失去启动空间 | 第一场必须按资源保全和第二场续航共同验收 | “先赢第一场再说”是合格计划 |

## A10 的主要重复失败

### 1. 加牌速度超过牌组的承载能力

- 各阶段失败样本普遍很少跳过奖励；随着推进，新增牌持续增加，但移除、升级、抽牌和能量没有同比增长。
- 问题不是固定张数，而是有效密度下降：基础牌、固定灾厄、慢能力和条件组件共同占用关键回合手牌。
- 结论：第一层前半允许补独立战斗力；完成前置输出后，跳过应成为默认，新增牌必须修复已命名缺口。

### 2. “有协同”被误当成“已形成系统”

- 多副牌同时拥有自伤、能力、消耗、力量或格挡组件，但核心错开上手时不能转化成当回合攻防。
- `能力 + 触发牌` 是一条依赖链，不是两条防线。继续添加触发器会提高上限，却可能进一步降低启动率。
- 结论：第一层末确定一个主要伤害成长和一个主要防御引擎；第二层不再开启新的慢主轴。

### 3. 休息税掩盖升级与稳定性缺口

- 早中期失败经常用休息修复上一场掉血，同时缺少能改变启动或阈值的核心升级。
- 部分 Boss 失败以高生命入场，说明更多回复并不能替代访问、独立防御或斩杀时钟。
- 结论：连续需要休息时，修正路线风险和构筑密度；不能把下一次休息当成牌组稳定性的组成部分。

### 4. A10 双 Boss 被按单 Boss 构筑

- 最远 A10 样本已经证明牌组能击败一个第三层 Boss，但第一战消耗后无法承受第二战早期压力。
- 另一类终局牌组满血进入首 Boss 仍在长战中失控，说明终局既缺启动效率也缺收束速度。
- 结论：首 Boss 的合格线是“胜后仍保留第二战有效生命、启动和保险”，不是单纯击杀。

### 5. 机制查询太晚

- 低生命自伤、回合开始触发、限制出牌、永久减益、复活/接续和随机目标都曾改变生死或构筑价值。
- 结论：陌生机制第一次影响不可逆决策时就查证。`playable=true` 与只覆盖敌方攻击的预览都不是生存证明。

## 低进阶胜局的正确用法

- 低进阶胜局说明“访问/能量能够让厚牌组运转”“消耗或攻击可以转成防御”“单一力量/易伤成长可以形成时钟”等协同具有功能价值。
- 它们没有证明这些协同在 A10 的数值、坏牌序、永久减益和连续双 Boss 下仍然够用。
- 因此只把相关卡组视为候选骨架。A10 当局若缺少独立即时防御、访问冗余、机制覆盖或双 Boss 预算，应放弃复刻而按缺口转向。

## 当前最值得检验的假设

不是“照抄低进阶胜局”，而是检验以下更严格命题：

> 在第一层末收敛主轴、显著提高跳牌门槛，并让压缩/访问、即时防御、成长防御和单一伤害时钟互相转化，能否减少 A10 的坏手死亡；进入第三层后，再以双 Boss 剩余资源而非单场胜率验收终局强度。

消耗防御、攻击转格挡与力量/易伤是目前可优先观察的组合，但尚未被 A10 胜局验证。若奖励不自然支持，不能强行追逐。

## 后续复盘格式

每个完整 A10 终局只记录：

1. 游戏/Mod/协议版本、角色与进阶。
2. 终局阶段与遭遇类型，不记录 run ID 或种子。
3. 四项验收在入场前是否通过：启动、防御、时钟、匹配。
4. 第一层末的主要伤害成长、防御引擎和访问方案是否已收敛。
5. 最早显著降低完整通关概率的不可逆决策。
6. 失败分类：机制误读、执行、构筑、路线、资源时机、技术异常或正常方差。
7. 只提出一个可在下一批 A10 尝试中检验的规则变化。

不要记录逐回合出牌、具体种子、可识别日志标识或依赖隐藏牌序的事后路线。终局具体操作没有可复现上下文时，不进入 skill。
