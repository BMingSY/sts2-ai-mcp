# STS2 关键遭遇与机制笔记

本文件只保存会改变路线、构筑或战斗决策的遭遇机制。读取后仍要查询当局 live state；A10 数值、Boss 身份、power amount 和选牌效果不得从旧样本硬套。

> 最近审阅：2026-07-22；机制数据基线为游戏 v0.107.1。

## 跨遭遇通用机制

- 多段攻击、荆棘、自动攻击和重放必须按实际每段/每次 play 的 live 结算计算。不要把“一张牌”默认成一次反伤。
- 复活、接续、召唤和爪牙会改变目标价值。击杀可复活单位的收益是本轮减伤与争取时钟，不等同于永久移除。
- 伤害上限、无敌、钻地和转阶段会让后续计划失效；跨过阈值或打掉最后一点特殊格挡后立即重读状态。
- `incoming_damage` 主要覆盖已暴露攻击意图。回合末 power、自伤、沙坑、紧缠和脚本触发必须另外读取。
- Boss/精英的基础伤害不是 A10 实际伤害。每回合用 runtime intent，不用本文数字做 lethal 计算。

## 第一层

### `SKULKING_COLONY` / 鬼祟珊瑚群

- `[当前机制 v0.107.1]` `HARDENED_SHELL_POWER` 会限制一回合可损失的 HP。超过上限的面板爆发不能继续压血。
- 把战斗规划成多个输出回合，并同步处理其来伤；不要把一次超额攻击当作斩杀。

### `TERROR_EEL` / 骇鳗

- `[当前机制 v0.107.1，A10 实战]` 已观察到 18 点单击，随后 4×3 并获得 6 活力，再进入 24 点强化单击。固定数字只用于解释样本；行动时仍以 live intent 和 power 为准。
- `SHRIEK_POWER` 会在生命第一次降到 75 或以下时触发眩晕。把跨过阈值视为一张能取消敌方行动的防御牌，并计算最晚必须在哪次强化重击前触发。
- 不要只比较当前回合格挡与掉血。若少打防御、先挂易伤或多打一张攻击能提前眩晕，应该比较支付的当前生命与被取消的整次攻击及后续安全输出窗。
- `[观察]` 过度格挡可能让牌组在强化攻击前仍未跨过眩晕阈值，最终只能依赖后续牌序命中多张防御。更早用可控生命换阈值进度，通常比保住较高面板 HP 后赌牌序更稳定。
- 触发眩晕后立即重读 intent；不要未经 live state 证明就假设眩晕、减益与下一次攻击之间固定有几个安全回合。

### `CEREMONIAL_BEAST` / 仪式兽

- `[观察]` 跨过其生命阈值会进入眩晕并清除已获得力量；阈值和触发条件在行动前以 live state/trace 验证。
- `[当前机制 v0.107.1]` `BEAST_CRY_MOVE` 后的限制 power 会压缩下一回合可打牌数。把唯一或有限额度留给最高价值防守、关键能力或斩杀牌。
- 不只按当前攻击防守；需要提前为受限回合保留格挡、牌顶或药水。

## 第二层

### `SLITHERING_STRANGLER` / 蛇行扼杀者

- `[观察]` `CONSTRICT_POWER` 会在玩家回合结束造成额外伤害并可叠层，敌人死亡后移除。
- 把紧缠单列进 end-turn 预算；双怪战通常优先处理扼杀者，避免层数继续增长。

### `TUNNELER` / 地道虫

- `[当前机制 v0.107.1]` `BURROW_MOVE` 获得特殊格挡并进入钻地循环，随后准备高伤攻击。
- `[观察]` 清空其钻地格挡会移除状态、取消攻击并转入眩晕。把打掉最后一点格挡视为防守动作；用低成本/多段牌剥层后再交高伤牌。

### `THIEVING_HOPPER` / 偷窃草蜢

- `[观察]` `SWIPE` 会暂时带走一张永久牌；击杀后通过奖励页手动取回。逃跑倒计时结束前必须完成击杀，不能假设牌自动返还。
- `FLUTTER` 按 hit 消耗并压低对应伤害时，先用低成本多段剥层，再让高伤单段吃完整倍率；第一击后重读。

### `DECIMILLIPEDE` / 残杀千足虫

- `[当前机制 v0.107.1]` 三个部位独立行动，死亡部位进入 `DEAD_MOVE -> REATTACH_MOVE`。
- `[观察]` 只要其他部位存活，死亡部位会在短倒计时后以部分生命接续。稳定打法是同步压血并在接续窗口内连续清场。
- 进入前需要连续多回合防御或真实多目标/定向减员能力；一次高格挡不能证明准备充分。

### `THE_OBSCURA` / 胧光怪 与 `PARAFRIGHT` / 寄生惧魔

- `[当前机制 v0.107.1]` 胧光怪会召唤寄生惧魔；寄生惧魔带 `ILLUSION_POWER`，死亡后下一回合满血复活，并带爪牙效果。
- 击杀幻象只在避免的当轮伤害足以抵消浪费的输出时划算。首领接近斩杀时优先结束首领，爪牙随之离场。
- 随机自动攻击可能打到复活幻象。不要在幻象即将回来时，把必须命中首领的高价值“打击”牌置顶给随机引擎。

### `SPINY_TOAD` / 棘刺蟾蜍

- `[当前机制 v0.107.1]` 招式循环为 `PROTRUDING_SPIKES_MOVE -> SPIKE_EXPLOSION_MOVE -> TONGUE_LASH_MOVE`。竖刺后获得荆棘，随后高伤爆发，再进入舔击。
- 荆棘按 live power 和实际命中结算。多段、重放和自动攻击会扩大反伤；无荆棘窗口要主动转化进攻资源，避免被循环拖死。
- 可保留格挡能跨过爆发回合，但覆甲等递减资源会随长战衰减。不能只防守而没有明确斩杀时钟。
- `[观察]` 把重放药水留到牌组已没有足够能量或抽牌转化时，无法再压低战斗尾部。应在较早的无荆棘窗口用药压缩战斗，而不是为未来 Boss 保留当前救命资源。

## Boss 与后期遭遇

### `THE_INSATIABLE` / 无厌沙虫

- `[当前机制 v0.107.1]` 会通过 `LIQUIFY_GROUND_MOVE` 加入大量状态牌，并在多段攻击、单段重击和强化之间循环。
- `[观察]` `SANDPIT_POWER` 可能产生 `incoming_damage` 未表达的失败条件。amount 接近危险阈值时，除非当回合击杀，否则优先保留并支付 `FRANTIC_ESCAPE` 的当前实际费用。
- `[待复验]` 沙坑、逃脱费用和失败顺序必须从当局 power、手牌与 trace 确认；旧样本的固定费用和伤害不可复用。

### `KNOWLEDGE_DEMON` / 知识恶魔

- `[当前机制 v0.107.1]` `CURSE_OF_KNOWLEDGE_MOVE` 会打开特殊选牌；只从当前 `selection.cards[]` 选择，并读取 runtime amount。
- 已观察的负面效果包括：持续自伤、少抽牌、限制每回合出牌数、少能量。不能按文字直觉排序，要按当前牌组的真实抽牌、期望出牌数、返能和剩余斩杀回合比较。
- `SLOTH` 类出牌上限会同时压缩 0 费、抽牌、返能和消耗链；低费链式牌组通常尤其怕它。
- 持续自伤在低血且没有明确短斩杀时风险极高。静态卡面基础值不能替代 A10 runtime 数值。
- `[观察]` 多次 `PONDER` 的治疗与加力量会让慢牌组失去竞速。进入战斗前需要确保主引擎能在首个危险循环前部署，并有明确斩杀时钟。

### `QUEEN` / 女王

- `[当前机制 v0.107.1]` 女王具有多段攻击、长期减益和手牌限制机制；`CHAINS_OF_BINDING_POWER` 一类效果会让出牌顺序改变后续可打牌。
- 每次出牌后检查 `unplayable_reason`。关键防牌、关键斩杀牌和能解除失败状态的牌优先；可选升级、低收益能力和试探牌后打。
- `TORCH_HEAD_AMALGAM` 会持续增加战斗压力。目标是尽快定向处理，而不是让随机伤害拖到多个成长循环。
- `[观察]` 延迟处理火炬头会让牌组无力追上女王成长；应准备不依赖随机目标的早期定向输出，但不设置脱离当局状态的固定回合阈值。
- 双 Boss 目标下，女王战需要稳定即时防御、减益后的防御冗余、定向爆发和长战成长；仅有一套能力上限不够。
- 自动复活药水面对多段攻击时逐 hit 结算；触发后仍可能继续吃剩余段数。

### 其他第三层 Boss

- 使用地图 `boss_identity` 和 live lookup 分别准备两个 Boss。本文没有记录的 Boss 不得套用女王或知识恶魔经验。
- 在最终火堆前检查两个 Boss 的共同需求与冲突需求，并为各自最危险回合分配药水；仍以第一场能稳定存活为前提。

## 事件与持续效果

### Wellspring 沐浴

- `[观察]` 移除基础牌后会加入临时 `GUILTY` / 愧疚，并按战斗倒计时移除。它是短期有效手牌损失换长期删牌，不是立即净化。

### Amalgamator 防御合并

- `[观察]` 两张普通防御合并为一张高质量防御时，既提高单牌质量又净减牌组；具体选项与数值以事件 live state 为准。

### 特殊选择页

- 事件/Boss 特殊选牌只选择 `selection.cards[]` 中明确合法的对象。不要从永久牌组猜选项，也不要用旧样本数值补 runtime 空缺。
