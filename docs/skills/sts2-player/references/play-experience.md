# STS2 NOSL A10 稳定通关策略

本文件只保存会直接提高完整通关率的策略。目标不是“当前血量最好看”，而是在不存读档重试（NOSL）的前提下，最大化击败第三层连续两个 Boss 并获得真实胜利结算的概率。

> 最近审阅：2026-07-20
> 验证基线：游戏 v0.107.1；Mod 0.1.7；protocol `2026-07-18-v2-draft`；state 13；decision 6。
> 数值、招式和卡牌文本以当局 live state 与 `lookup_game_data` 为准。本文件不提供固定卡牌评级表。

## 目标函数与 NOSL 边界

- 只有在第二个第三层 Boss 后出现真实胜利结算，才算完成一局。到达 Boss、击败第一个 Boss 或高血进入第三层都不是成功。
- NOSL 意味着不通过保存、退出、重载、回滚或调试接口重新抽取事件、牌序、奖励、目标或战斗结果。正常暂停不应被用来重试已知随机结果。
- 不读取 live state 明确隐藏的抽牌顺序、事件结果或随机目标。决策只使用当时合法可见的信息；复盘也要按当时可知信息评价，不能用结果倒推“唯一正确答案”。
- “最稳”指最大化完整通关概率，不等于最少掉血。HP、药水、金币和升级都是把当前风险换成后续强度的资源。
- 连胜目标惩罚低概率崩盘。两条平均收益相近的路线，优先选择最坏常见结果仍可继续运行、且保留转向空间的一条。

## 如何衡量“最稳”

每个不可逆决策都按下表检查，不用单一 HP 或牌组张数代替判断。

| 维度 | 要回答的问题 | 常见危险信号 |
| --- | --- | --- |
| 当前生存 | 这个房间及下一个强制房间的常见坏牌序能否处理？ | 只靠一张牌、一次随机目标或战后治疗才能活 |
| 启动稳定性 | 核心攻防能否在需要前上手并支付？ | 能量很多但手牌不可转化；能力沉底即失效 |
| 终局强度 | 能否同时处理成长 Boss、长战和爆发回合？ | 只会拖延，没有斩杀时钟；或只会爆发，没有续航 |
| 匹配覆盖 | 是否有多目标、减益、荆棘、伤害上限、复活/接续等答案？ | 单一随机攻击引擎被一个机制完全关闭 |
| 资源与选项 | 路线是否留有火堆、商店、药水和转向机会？ | 为未来奖励耗尽当前保险；连续未知/强制战无恢复出口 |

决策顺序：

1. 先排除当回合或当前房间的确定死亡线。
2. 再比较未来一个抽牌循环与后续 2-3 个节点的下行风险。
3. 在都能稳定存活的线路中，选择更能通过本层 Boss 与第三层双 Boss 检查的一条。
4. 只有生存、启动和终局强度都已覆盖，才追求额外 HP、金币或理论上限。

不要把“有可能抽到答案”算作已经覆盖。如果一个威胁只有一张牌能解，必须同时有可靠抽牌、保留、牌顶控制或药水冗余，否则仍是单点故障。

## 构筑纪律

### 按功能而不是按卡名构筑

每张新增牌至少解决一个当前缺口，并说明它在最坏常见手牌中的作用。

| 功能 | 稳定通关需要的表现 |
| --- | --- |
| 前置输出 | 早期能快速减少敌人数或越过危险机制，不依赖先铺多张能力 |
| 即时防御 | 抽到即可支付并生效；至少有一类不依赖核心能力、随机消耗或未来回合结算 |
| 牌序与资源 | 抽牌、保留、牌顶、消耗压缩和能量能互相转化，而不是各自孤立 |
| 多目标/目标控制 | 能降低多敌人总来伤；随机目标不能替代定向处理关键敌人 |
| 成长与斩杀时钟 | 能在长战持续提高攻防，或在敌人失控前可靠结束战斗 |
| 机制答案 | 弱化、降力量、净化、穿透上限、同步击杀、处理复活等按当局 Boss 补齐 |

### 新牌默认需要证明自己

- “跳过”是正常选项，但不是教条。当前缺少终局成长时，即使下一战收益一般，也要拿能实质提高 Boss 胜率的成长牌。
- 只因“有协同”不能加牌。新牌必须提高启动速度、功能冗余、牌流、关键回合强度或明确的 Boss 匹配。
- 额外卡牌奖励增加的是选择权，不是拿牌配额。即使遗物让普通战出现多组奖励，也要对每一组独立执行拿牌门槛；不能因为“奖励免费”就同时拼多个半成品组件。
- 已知 Boss 若按伤害实例数限制伤害（例如有限次数的伤害压缩、无敌层数或需要逐次击破的保护），多段牌的价值要按“每费可触发几次有效命中”重估。缺少这类机制答案时，一张中等面板、多次命中的牌可能优于又一张高单段攻击。
- 下一节点是强制战时，提高该战坏牌序生存率的牌优先；下一节点安全且终局强度不足时，可以为长期成长付出小幅即时成本。
- A10 的固定死牌会降低有效手牌密度。额外诅咒、任务牌和不可打牌必须有可兑现且足够大的收益，并准备消耗、移除或额外抽牌补偿。

### 不用牌组张数替代密度

- 厚牌组不是自动失败，薄牌组也不是自动稳定。评估的是有效牌密度、核心访问率、费用曲线和战斗内压缩能力。
- 厚牌组需要稳定抽牌、能量、0费牌、保留、牌顶或消耗引擎作补偿；没有补偿时，每张边际牌都会降低关键牌及时出现的概率。
- 任务牌、固定灾厄和敌人塞入的状态牌会叠加成同一种失败模式：关键回合有效手牌不足。评估未来高收益任务时，先把它在当前 Boss 前每个抽牌循环占用一次手牌的成本计入；只有活过当前层，未来金币或遗物才有价值。
- 不要同时拼多个半成品主轴。选择一个主引擎后，其他牌应补它的启动、防御、牌流或匹配，而不是再开新包。

### 依赖关系不能伪装成多条防线

- `能力 + 触发牌` 算一条依赖链，不算两条独立防线。
- 即时格挡、弱化/降力量、攻击转格挡、消耗格挡、覆甲/敏捷成长可以组成多层防御，但至少一层要在核心能力沉底时仍能工作。
- 自动攻击、重放和随机目标引擎必须检查荆棘、伤害上限、复活部件和优先目标。某一普通怪能完全关闭引擎，说明它不能作为唯一主轴。
- 能量只有能转化为当回合有效牌时才有价值。选能量遗物或返能牌前，检查手牌密度、抽牌和费用，而不是只看总能量。

### Boss 奖励按稳定性而不是上限选择

- 稀有牌和 Boss 遗物要问：坏起手能否使用、是否需要另一张牌才能工作、是否扩大特定机制弱点、是否损害药水/能量/抽牌的下行尾部。
- 给慢能力降费只有在该能力确实是可靠主轴时才高价值。不要因为已经投入一次升级或奖励而继续追加沉没成本。
- 战后治疗只在赢下当前战斗后存在，不能计入当前 HP。依赖战后回复的构筑更需要确保每场战斗第一循环不会被锁死。

## 路线纪律

### 没有固定房间优先级

- 普通战是较可预测的金币、卡牌和药水成长来源；未知房是高方差选项。不能默认“事件优于普通战”。
- 精英不是单纯风险，也提供通关所需的遗物成长。完全躲精英可能让牌组以健康但强度不足的状态死于双 Boss。
- 火堆不是自动休息点；商店也只有在金币和库存能修复当前缺口时才有价值。

### 选路时看完整约束，再重点看近期

1. 先看本层 Boss、所有可达火堆/商店、强制精英和路线汇合点。
2. 对每条候选路线追踪到下一个可靠恢复点或真实转向点；记录这段内的必经战斗、可能转为战斗的未知节点、强制精英、可退出分支，以及该检查点之后立刻锁定的精英或战斗。商店库存未知，不能自动视为恢复点；早期商店也不能遮住前后连成一段的高密度风险。
3. 再重点比较后续 2-3 个节点的连续风险，尤其是未知转战斗后紧接强制战、低血无恢复、或商店金币不足。
4. 未知节点不按“必定战斗”机械计数，也不能只按极端最坏结果选路；但如果一个并不罕见的战斗尾部会在恢复前耗尽 HP/药水，且路线没有转向，它就是路线自身的失败概率，不能事后归为单纯运气。
5. 保留能根据下一次奖励转向的分叉；不要过早锁进只有一种房间序列的路线。一次火堆也不能自动洗掉后续连续强制精英的资源压力。
6. 选路理由必须同时说明“如何活过整个连续风险段”和“如何获得双 Boss 所需强度”，不能只证明下一场能打或某个商店收益很高。

### 精英采用准备度检查

进入精英前至少确认：

- 当前已知精英池的关键机制有答案；
- 前两回合既有输出也有可支付防御，不依赖完美牌序；
- 有药水或其他保险覆盖一个坏回合；
- 打完后的路线允许恢复、商店修复或安全转向；
- 遗物收益确实能提高后续完整通关率。

HP 高不能替代这些检查，HP 低也不自动禁止精英；关键是最坏常见结果是否仍能继续运行。

### 火堆按边际通关率选择

- 休息：当回复能跨过下一段路线或 Boss 的死亡区间，并且当前牌组已有足够终局强度。
- 锻造：当升级能显著提高核心访问、费用、防御阈值、清场或斩杀时钟，且不休息仍有可靠路线。
- 其他选项：移除、变化等要与当前最主要失败模式比较，不按固定百分比触发。
- 不设“低于 70% 必休息”之类硬阈值。满血弱牌组不稳，较低血但能可靠结束战斗的牌组可能更稳。

## 资源纪律

### HP

- HP 是风险预算，不是分数。可用来换取精英遗物、关键升级和更强路线，但不能把一次低概率崩盘包装成“用血换成长”。
- 自伤、荆棘、回合末伤害和敌方攻击分别计入预算。只有尚未被消耗的格挡或明确触发的效果可以抵扣。
- 自伤引擎要在 Boss 前做“有效生命审计”：把预计回合数内的回合开始自伤、主动自伤、无法避免的蹭血和一次峰值攻击同时计入。若一次自伤会让 HP 低于当前可见攻击，只在它能产生确定斩杀或确定防御时支付；单纯提高随机斩杀概率不算安全转化。
- 低血时停止拿只提高理论上限、却不提高下一场第一循环生存率的边际牌。

### 药水

- 药水的首要作用是压低失败尾部，不是留到 Boss 展示收藏。当前战斗可能终结运行时，应在仍有确定转化窗口时使用。
- 不要等到“只有药水高掷才能活”才喝。若药水能把危险战斗从依赖牌序变成稳定线路，通常应提前用。
- 若当前已有安全的信息揭示动作（抽牌、选择牌或不会改变生存线的生成效果），通常先完成它，再决定是否喝敏捷、格挡或攻击药水。不要在尚未看完本回合确定可见的牌流前，提前消耗一瓶可能被随后斩杀线证明为不必要的保险。
- 外部抽牌药水与“本回合不可再抽牌”效果要按顺序结算：需要两者时，先用药水，再打出封锁后续抽牌的牌。评估抽牌药水时要看它能否把一手死牌扩展成第二次找防御/斩杀的机会，而不只看平均抽牌数。
- Boss 前为两个 Boss 分配药水，但第一场若不喝就可能死，必须先保证进入第二场。
- 自动攻击/重放药水要在无荆棘、目标明确且仍有能量/抽牌转化时使用；不要在引擎已经被迫关闭后才补救。
- 药水栏满时比较的是每瓶能覆盖的具体失败模式，不按稀有度机械保留。用一次性抽牌、敏捷或减伤药水交换永久最大生命前，要比较“多出的有效生命”与“药水能否覆盖一次致死坏牌序”；少量面板生命通常不能替代后者。

### 金币与升级

- 商店先修当前最大单点故障：关键移除、即时防御、稳定抽牌、药水保险或主引擎组件。
- 便宜的边际牌仍会占抽牌位；买不起真正修复项时，保留金币通常优于凑消费。
- 升级优先考虑费用变化、抽牌、关键防御阈值和主引擎启动，不只看伤害增量。

## 战斗纪律

### 每回合的判断顺序

1. 汇总敌方攻击、回合末 power、自伤、荆棘/反伤、多段结算、复活和自动药水；不要只看 `incoming_damage`。
2. 检查确定斩杀、击晕、转阶段或击杀关键目标能否消除更多未来伤害。
3. 找到本回合确定生存线，再判断抽牌或随机效果是否值得替换它。
4. 比较本回合 HP 损失与未来一整个循环的风险，不只追求当回合无伤。
5. 结束回合前重算一次能量、格挡、未建模触发和下回合保留资源。

### 稳定线路优先

- 已有确定生存线时，不为了小幅期望收益把它改成随机抽牌、随机目标或随机消耗线路。
- 当前确定防不住时，主动寻找击杀、击晕、药水、抽牌和牌顶的逃生线；这时赌线才合理。
- 抽牌动作前明确剩余能量与“抽到什么才有用”。抽到高费答案却无法支付，不是有效出口。
- 可保留格挡只计算实际保留上限；为了预存格挡付出的 HP、能量和溢出都要计价。

### 目标与反伤

- 多目标战优先减少总来伤，但可复活/接续单位要比较“本轮避免的伤害”与“浪费的斩杀输出”。
- 随机目标能力不能承担必须命中首领或关键小怪的任务。牌顶控制触发随机自动攻击前，要把可能打错目标列入成本。
- 荆棘按每个实际命中或每次引擎重放的 live 结算计算。能斩杀或避免更大伤害时可以攻击，不能机械规定“荆棘回合永不攻击”。
- 伤害上限、无敌和转阶段会让面板爆发失真；第一击后重读状态，再提交后续攻击。
- 对有限次数的伤害压缩/免疫层，先用低费多段、0 费攻击和低价值单段牌剥层，再投入高伤单段、易伤窗口和有限重放资源。还要通过 live move/power 确认敌人的准备动作是补层、加力量还是其他效果；不能凭名称假定护盾会刷新或不会刷新。

### 同步击杀、朝向与随机目标

- 面对“队友死亡后存活者大幅强化”的多首领战，把它视为同步击杀检查。击杀第一个目标前，先计算剩余定向与 AOE 能否处理强化、格挡和下一个意图；不能只看当前两边 HP 是否相等。
- 遭包围时，把“最后一张有目标的牌或药水决定朝向”纳入整个回合顺序；无目标 AOE 不要误算为会修正朝向。优先消除当回合与下回合的确定死亡，再追求对称压血。
- 使用随机目标攻击前，枚举所有有意义的落点分布。只有每种分布都保留生存或同步斩杀时，才把它计为稳定线；若必须在 n 段中命中指定目标至少 k 次，明确将其标记为赌线，并向前寻找非随机分歧。
- `[当前机制 v0.107.1]` 不把致死命中后的荆棘/反伤计为生存资源。`7J7STSZTLH` 中玩家 15 HP 承受 17 伤害、铜鳞随后击杀 3 HP 攻击者，游戏仍先进入角色死亡结算。除非 live 机制明确证明反伤先于死亡，否则先要求玩家存活过该次命中。

### 不把终局锁死误判成最后一手失误

如果当前回合已无生存线，记录终局状态后结束战斗；复盘要寻找更早的分歧点，例如：

- 药水使用晚了一个循环；
- 主引擎被当前怪物完全克制；
- 进入连续战斗前没有即时防御覆盖；
- 为战后治疗、未来 Boss 或理论协同保留了当前必须使用的资源；
- 牌组能量、抽牌和费用无法同时转化。

## 分阶段检查表

这些是问题清单，不是硬数值门槛。

### 第一层前半

- 是否已补足能快速减少敌人数的前置输出？
- 第一只精英的坏起手能否用基础资源和药水覆盖？
- 新牌是否仍解决当前缺口，而不是提前拼慢速终局包？

### 第一层 Boss 前

- 能否处理已知 Boss 的限制、伤害上限、召唤或转阶段？
- 若限制按命中次数解除，当前牌组每轮能以多少能量制造多少次伤害实例？清除限制后，是否仍剩能量或下一回合易伤层数来兑现核心输出？
- 是否有至少一条即时防御和一条后续成长/循环防御？
- Boss 奖励会不会让第二层第一循环更慢或更依赖随机目标？

### 第二层

- 是否有真实多目标答案或快速定向减员能力？
- 是否能连续处理多个中高伤回合，而不是只挡一次峰值？
- 核心能力沉底时，前 5-10 张牌是否仍有可支付攻防？
- 路线是否避免低血连续未知/强制战，同时仍获得足够成长？

### 第二层 Boss 前

- 已知 Boss 的特殊失败条件是否由 live power/selection 明确解释？
- 是否有斩杀时钟，避免敌人治疗、加力量、塞状态或沙坑等机制拖到失控？
- 关键能力能否较稳定地在 T1-T4 上场，或有不依赖它的替代线？

### 第三层与连续双 Boss

- 牌组能否分别处理前置爆发、多目标/关键小怪和长战成长，而不是只针对一种 Boss？
- 第一轮坏牌序是否有冗余防御、抽牌、保留或药水兜底？
- 关键牌被锁、弱化、脆弱或限制出牌数时，是否还有第二条攻防转化线路？
- 两场 Boss 的药水和一次性资源如何分配？第一场的必要消耗不能为了第二场被错误保留。
- 第一个 Boss 结束后即使资源大幅下降，是否仍有能力启动第二场？

## 复盘与维护

### 复盘顺序

1. 记录死亡/胜利时的可见状态，判断终局是否仍有合法生存或斩杀线。
2. 向前寻找最早一个显著改变完整通关概率的不可逆决策，不停留在最后一次抽牌。
3. 区分：规则误读、执行错误、构筑单点故障、路线风险、资源时机，以及正确决策后的正常低概率失败。
4. 用当时可知信息做反事实；不能因为后来知道抽牌顺序，就声称先前应按隐藏顺序行动。
5. 只把跨构筑可复用的结论写回本文件；具体局保存在 `run-retrospectives.md`。

### 状态标签

- `[原则]`：用于 NOSL A10 稳定通关的通用决策规则。
- `[当前机制 vX]`：由对应游戏版本数据或 live state 验证；版本变化后复核。
- `[观察 run ID]`：单局证据，只能用于提出或支持假设。
- `[待复验]`：机制或结论仍不完整，行动前必须查询 live state。

维护时合并重复结论，删除已修复且没有操作价值的 MCP 历史问题。不要把开发回归记录、完整战斗流水或静态卡牌说明重新塞回本文件。
